# Flow Diagram

## Purpose

This diagram explains the current MVP wedge in one glance.

## Current wedge

Verified Work / Remote Worker TrustPass

## Main steps

1. Partner launches trust-sensitive flow
2. User completes proof / review step
3. PassPal requests minimum useful proof
4. PassPal evaluates workflow inputs
5. PassPal returns decision receipt
6. Partner chooses next action

## Possible next actions

- Continue
- Ask for one more proof
- Escalate
- Stop

## Short interpretation

The flow is designed to help the relying party move from proof request to decision-ready action.
The goal is not just identity collection.
The goal is a clearer trust outcome.
