# Remote Worker Flow

## Flow summary

1. A partner launches a trust-sensitive flow
2. A user completes the proof or review step
3. PassPal requests the minimum useful trust inputs
4. PassPal evaluates the workflow
5. PassPal returns a receipt
6. The partner decides what happens next

## Possible next actions

Depending on the result, the partner can:

- continue
- ask for one more proof
- escalate
- stop

## Product value

The point of the flow is not only to verify identity.

The point is to help the relying party reach a clearer trust decision.
